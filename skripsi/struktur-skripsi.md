#### 2.1.7 Pengertian Black Box Testing (2 halaman)

**Referensi yang digunakan:**

1. **Ostrand (2002)** - "Black-Box Testing"
   > "Black-box testing of software is any method of generating test cases that is independent of the software's internal structure. Since black-box testing is always based, either directly or indirectly, on the software's specification, it is also called specification-based testing." [31]

2. **Khan (2011)** - "Different Approaches To Black box Testing Technique"
   > "Black Box Testing is a method of generating test cases that are independent of software internal structure. Various black box testing approaches include model-based, scenario-based, data-based, statistical, and random testing." [32]

3. **Priyaungga et al. (2020)** - "Pengujian Black Box pada Aplikasi Perpustakaan"
   > "Black Box Testing adalah pengujian berdasarkan tampilan (interface) dan fungsi software itu sendiri dan bukan dari source code program. Teknik Equivalence Partitions membahas valid atau tidaknya yang dimasukkan ke dalam software." [33]

4. **Jaya (2018)** - "Pengujian Aplikasi dengan Metode Blackbox Testing Boundary Value Analysis"
   > "Pengujian menggunakan black box testing Boundary Value Analysis. Boundary Value Analysis adalah jenis test case dengan menentukan nilai normal, nilai minimum dan nilai maksimum dari data yang diuji." [34]

5. **Jacob & Prasanna (2016)** - "A Comparative analysis on Black box Testing strategies"
   > "Black box testing methods play a crucial role in ensuring quality software development, with 30% effort spent on testing activities in the Software Development Life Cycle." [35]
